Quick Start

# One-line deployment
curl -sSL https://tada.wang/install.sh | bash -s -- create

# Initial setup
curl -sSL https://tada.wang/install.sh | bash -s -- setup

No Docker Required!

The deployment tool runs directly via uvx (Python package runner) - no Docker installation needed on your machine!

Installation

The installer automatically handles everything for you:

1. Initial Setup

curl -sSL https://tada.wang/install.sh | bash -s -- setup

This will create a config.yaml file in your current directory

2. Configure

Edit the configuration file with your settings:

nano config.yaml

3. Deploy

curl -sSL https://tada.wang/install.sh | bash -s -- create

Commands

  • setup - Initial configuration setup
  • create - Create spot instances
  • list - List running instances
  • destroy - Destroy all spot instances
  • help - Show help message

Architecture

On Your Machine:

  • No Docker required - runs via uvx
  • Automatic dependency management
  • Direct execution from GitHub

On EC2 Instances:

  • Bacalhau: Runs as Docker container (ghcr.io/bacalhau-project/bacalhau:latest-dind)
  • Sensor Generator: Runs as Docker container (ghcr.io/bacalhau-project/sensor-log-generator:latest)
  • Docker is automatically installed via cloud-init

AWS Authentication

The tool automatically detects AWS credentials from:

  • Environment variables (AWS_ACCESS_KEY_ID, etc.)
  • AWS SSO sessions - just run aws sso login first!
  • AWS config files (~/.aws/credentials)
  • EC2 instance roles
Pro Tip: The installer will tell you exactly which credentials it's using!

Advanced Usage

Direct uvx Usage

# Run directly without the installer
uvx --from git+https://github.com/bacalhau-project/aws-spot-deployer spot-deployer create

Local Installation

# Clone and use locally
git clone https://github.com/bacalhau-project/aws-spot-deployer.git
cd aws-spot-deployer
./spot create

Configuration Options

The config.yaml file supports:

  • Multiple AWS regions
  • Custom instance types
  • Dedicated VPCs per deployment
  • Tagging for cost tracking
  • SSH key management
  • Bacalhau orchestrator credentials
  • Custom scripts and commands
Security Note: Never commit your AWS credentials or private keys to version control!

Bacalhau Integration

To connect to a Bacalhau orchestrator, create these files before deployment:

mkdir -p files
echo "nats://orchestrator.example.com:4222" > files/orchestrator_endpoint
echo "your-secret-token" > files/orchestrator_token
